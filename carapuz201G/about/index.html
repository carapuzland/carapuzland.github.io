About Carapuz201G

Project overview, architecture, storage and feature philosophy

What is Carapuz201G?

Carapuz201G is a large multi-purpose Discord bot focused on combining server administration, moderation, utility tools, economy systems, verification, profile tools, reminders, giveaways, voice features and optional local AI features into a single bot.

The bot is developed and maintained as an independent project by a single developer. It is intended for practical use in Discord communities rather than as a simple demonstration bot.

How the Bot Works

The bot runs as a long-living Discord application process and reacts to:

  • slash commands,
  • text prefix commands,
  • message events,
  • voice state changes,
  • scheduled background loops such as reminders, backups, temp role checks and giveaway checks.

It stores configuration and internal state in encrypted local files and uses runtime memory for temporary data, queues, and active session state.

Project Architecture

The current public build is a large monolithic Python bot file using discord.py. The bot uses a central encrypted configuration store, language files for responses, optional local AI integration, voice support, and multiple UI systems such as buttons, modals and select menus.

The bot is intentionally designed as a highly integrated all-in-one system instead of splitting every feature into separate micro-bots.

Data Storage

Depending on the deployment environment, bot data may be stored:

  • on the operator’s local computer,
  • on a self-hosted server,
  • or on a hosting provider used to operate the bot.

Persistent data is encrypted at rest. Temporary files may also be created for voice or media processing.

Optional Local AI

Some builds of Carapuz201G may support local AI processing through Ollama.

  • AI usage is intended only for feature execution.
  • Discord user messages are not used to train or fine-tune models.
  • Any local context history, if enabled, is intended only to preserve short feature context.
  • The project is designed to avoid training models on Discord content.
The intended AI model strategy is local-only processing whenever AI is enabled.

Voice and Media

The bot may use voice features such as music, speech, countdowns, and radio-style playback. These features depend on the current host environment and may be limited or disabled if the host does not support them reliably.

Why This Bot Exists

The goal of Carapuz201G is to give Discord communities a single powerful assistant for:

  • moderation and recovery,
  • server automation,
  • economy and game features,
  • verification and user tools,
  • voice and utility features.
